第2课：多层感知机与反向传播

把神经元堆起来，再看"学习"到底怎么算出来的
预计时间：30 分钟  |  2026-06-17

1. 回顾：一个神经元的局限

上节课我们知道了：一个神经元 = 线性模型。它能画一条直线（或高维超平面），但真实世界的数据不是线性的。

解决方案：把多个神经元叠起来，层与层之间加激活函数。这就是多层感知机（MLP, Multi-Layer Perceptron）。

输入层 隐藏层 1 隐藏层 2 输出层 (3维) (4个神经元) (4个神经元) (1维) x₁ ──────→ h₁⁽¹⁾ ──→ h₁⁽²⁾ ╲ ╱ ╲ ╱ ╳ ╳ ──→ ŷ ╱ ╲ ╱ ╲ x₂ ──────→ h₂⁽¹⁾ ──→ h₂⁽²⁾ ╲ ╱ ╲ ╱ ╳ ╳ (每条线 ╱ ╲ ╱ ╲ 都是一个 x₃ ──────→ h₃⁽¹⁾ ──→ h₃⁽²⁾ 权重 w) │ h₄⁽¹⁾ ──→ h₄⁽²⁾ 每个隐藏层: h⁽ˡ⁺¹⁾ = σ(W⁽ˡ⁾ h⁽ˡ⁾ + b⁽ˡ⁾) σ = 激活函数（如 ReLU）
图1：一个 3→4→4→1 的 MLP。h⁽ˡ⁾ 表示第 l 层的输出。
命名法："隐藏层"（hidden layer）这个名字来源于——你只看到输入和输出，中间层的值在训练数据里是看不到的，是"隐藏"的。LLM 里的 hidden_size（比如 4096）就是指每层有多少个隐藏神经元。

2. 正向传播：从输入到输出

用矩阵形式写，一个两层的 MLP：

h = ReLU(W₁x + b₁)     # 第1层：线性变换 + 激活
ŷ = W₂h + b₂            # 第2层：线性变换（输出层不加激活）

假设输入 x 是 3 维向量，隐藏层有 4 个神经元：

矩阵形状含义
W₁4 × 3每行是一个隐藏神经元对 3 个输入的权重
b₁4每个隐藏神经元的偏置
W₂1 × 4输出层对 4 个隐藏值的权重
b₂1输出偏置

正向传播就是做两次矩阵乘法 + 一次 ReLU。这就是 PyTorch 里 model(x) 在干的事。

3. 反向传播：Loss 怎么反向流动？

第 1 课我们用伪代码算过 w * x + b 的梯度。现在网络有 W₁, b₁, W₂, b₂ 四个参数，每个都需要算梯度。

核心工具是链式法则（chain rule）。你高数课上学过：

如果 L = f(g(x))，那么 ∂L/∂x = f'(g(x)) · g'(x) 推广到多层： L = loss( W₂ · ReLU(W₁·x + b₁) + b₂ , y_true ) ┌──────────────┐ ∂L/∂W₂ = │ ∂L/∂ŷ · ∂ŷ/∂W₂ │ (输出层梯度) └──────────────┘ ┌──────────────────────────┐ ∂L/∂W₁ = │ ∂L/∂ŷ · ∂ŷ/∂h · ∂h/∂z · ∂z/∂W₁ │ (隐藏层梯度) └──────────────────────────┘ ↑ ↑ ↑ 输出层 激活函数 线性变换 对 h 的 ReLU 的 对 W₁ 的 梯度 梯度 梯度
直觉：反向传播就是把 loss 这个"错误信号"从输出层逐层往回传。每一层根据自己收到的错误信号，调整自己的参数。离输出越近的层，信号越清晰；离输入越近的层，信号越模糊（这就是"梯度消失"问题的根源，Transformer 那课会讲怎么解决）。

4. 反向传播的五个步骤（手动跟踪一遍）

假设我们有一个最简单的两层网络，不用矩阵，只用一个标量来跟踪每一步：

已知：x=2, y_true=5 参数初始值：w₁=0.5, b₁=0, w₂=1.0, b₂=0 激活函数：ReLU(x) = max(0, x) Step 1 — 前向传播： z₁ = w₁·x + b₁ = 0.5×2 + 0 = 1.0 a₁ = ReLU(z₁) = max(0, 1) = 1.0 ŷ = w₂·a₁ + b₂ = 1.0×1 + 0 = 1.0 L = (ŷ - y_true)² = (1-5)² = 16 Step 2 — 输出层梯度（离 loss 最近，最好算）： ∂L/∂ŷ = 2(ŷ - y_true) = 2(1-5) = -8 ∂L/∂w₂ = ∂L/∂ŷ · a₁ = -8 × 1 = -8 ∂L/∂b₂ = ∂L/∂ŷ · 1 = -8 Step 3 — 隐藏层梯度（要把梯度"穿"过激活函数）： ∂L/∂a₁ = ∂L/∂ŷ · w₂ = -8 × 1.0 = -8 ∂L/∂z₁ = ∂L/∂a₁ · ReLU'(z₁) ReLU'(1.0) = 1（因为 z₁ > 0） = -8 × 1 = -8 ∂L/∂w₁ = ∂L/∂z₁ · x = -8 × 2 = -16 ∂L/∂b₁ = ∂L/∂z₁ · 1 = -8 Step 4 — 梯度下降更新（lr=0.01）： w₁ ← 0.5 - 0.01×(-16) = 0.66 b₁ ← 0 - 0.01×(-8) = 0.08 w₂ ← 1.0 - 0.01×(-8) = 1.08 b₂ ← 0 - 0.01×(-8) = 0.08
重要：实际训练中你不需要手算这些梯度。PyTorch 的 loss.backward() 自动完成 Step 2-3，optimizer.step() 自动完成 Step 4。但理解这个流程是读懂 Transformer 训练逻辑的前提。

5. PyTorch 中这长什么样

import torch
import torch.nn as nn

# 定义一个 3→64→1 的 MLP（就是两个 nn.Linear）
model = nn.Sequential(
    nn.Linear(3, 64),    # W₁: 64×3, b₁: 64
    nn.ReLU(),           # 激活函数
    nn.Linear(64, 1),    # W₂: 1×64, b₂: 1
)

# 训练循环 —— 注意和上节课伪代码一模一样
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

for epoch in range(100):
    y_pred = model(x)                     # Step 1: 前向传播
    loss = ((y_pred - y_true) ** 2).mean() # 算 loss
    optimizer.zero_grad()                  # 清空上一次的梯度
    loss.backward()                        # Step 2-3: 自动反向传播
    optimizer.step()                       # Step 4: 更新参数

这 5 行训练循环是所有深度学习框架的通用模式，从最简单的 MLP 到 GPT-4 都长这样。

8. 下节预告

MLP 的问题是：输入必须固定长度，而且"昨天"和"365天前"的信息权重完全独立——不适合处理序列。下节课我们进入Transformer 的核心：Self-Attention 机制，这是理解 LLM 的关键一跃。

关键要点：MLP = 线性层 + 激活，堆叠起来。反向传播 = 链式法则 + 自动微分。PyTorch 帮你做了反向传播，你只需要定义前向传播（forward）+ 写训练循环。
