第3课：Self-Attention —— Transformer 的灵魂

从"固定权重"到"动态权重"：让每个 token 自己决定要看谁
预计时间：35 分钟  |  2026-06-17

1. MLP 处理序列为什么不行？

你有训练经验，这条应该深有体会。假设要把一句话"央行降息利好债市"喂给 MLP：

MLP 方案：把所有词拼成一个大向量
x = [央行, 降息, 利好, 债市]  → 固定长度 embedding
h = ReLU(Wx + b)

问题1：句子可长可短，MLP 输入维度固定
问题2："降息"和"利好"的关系、「降息」和"央行"的关系
         ——MLP 对每对词的权重是固定学死的
问题3："利好"修改的是"债市"，离了 2 个词位
         —— MLP 无法建模长距离依赖

RNN/LSTM 解决了一部分（可变长度 + 时序传递），但"利好"要传到"债市"需要一步一步走过去，距离越远信息衰减越严重，而且无法并行计算。

2. Attention 的核心思想：动态加权

一句话总结 Attention：对于当前位置，扫一遍所有位置，算出每个位置跟当前位置的"相关度"，然后用这个相关度做加权平均。

句子： 央行 降息 利好 债市 ↓ ↓ ↓ ↓ 1 2 3 4 (位置) 假设你在"利好"(位置3)，要聚合上下文信息： 和"央行"的相关度：0.1 ← 有点关系但不是直接修饰 和"降息"的相关度：0.3 ← 降息是利好逻辑上的触发条件 和"利好"的相关度：0.5 ← 自己跟自己当然最相关 和"债市"的相关度：0.1 ← 利好修饰债市，但语义上没那么近 加权平均 = 0.1·v(央行) + 0.3·v(降息) + 0.5·v(利好) + 0.1·v(债市)

注意：这些"相关度"不是固定的——同一个词在不同上下文中，跟同一个目标词的相关度完全不同。相关度是根据当前输入内容动态算出来的。

3. Q、K、V：Attention 的三件套

怎么算出这个"相关度"？经典的 scaled dot-product attention 用了三步：

Attention(Q, K, V) = softmax( Q·Kᵀ / √dₖ ) · V

拆开来看：

矩阵含义怎么来的
Q (Query) "我在找什么？"——当前 token 的查询向量 x · W_Q，每个 token 有自己的 Q
K (Key) "我是什么？"——每个 token 的标签向量 x · W_K
V (Value) "我有什么信息？"——每个 token 的实际内容 x · W_V

计算流程：

1. Q @ Kᵀ     → (seq_len × dₖ) @ (dₖ × seq_len) = (seq_len × seq_len)
   每一对 token 之间的"原始相关度分数"（attention scores）

2. / √dₖ      → 缩放。dₖ=64 时不缩放，点积值可能到 ±20，
                  softmax 后会变成 one-hot，梯度趋近于 0

3. softmax()  → 每一行变成概率分布（和为 1）—— 这就是"注意力权重"

4. @ V        → 用注意力权重对 V 做加权平均
   (seq_len × seq_len) @ (seq_len × dᵥ) = (seq_len × dᵥ)
   每个 token 得到一个"看完全文后"的新表示
关键直觉：Attention 做的事 = 每个 token 发出一个 Query（"谁跟我有关？"），每个 token 挂一个 Key（"我是什么？"），匹配上的就把那个 token 的 Value（"我有的信息"）拉过来。整个过程完全可微分，所以可以端到端训练。

4. Multi-Head Attention：为什么不只用一个头？

如果只用一组 Q/K/V，一个词只能学会一种"关注模式"。但语言是复杂的——"降息"对"利好"的关注应该同时包含：

  • 语法关系（主谓/动宾）
  • 语义关系（因果/转折）
  • 位置关系（相邻/跨句）

Multi-head 的做法：把 d_model 切成 h 份，每份独立做 attention，最后拼回来。

d_model = 512, h = 8, dₖ = dᵥ = 512/8 = 64 Q: (seq, 512) → 切成 8 个 (seq, 64) K: (seq, 512) → 切成 8 个 (seq, 64) V: (seq, 512) → 切成 8 个 (seq, 64) 每个头独立算 Attention(Qᵢ, Kᵢ, Vᵢ) → (seq, 64) 8 个头的结果拼起来 → (seq, 512)，再过 W_O 投影

不同头确实会学到不同的模式——有的关注相邻词（局部语法），有的关注远距离主语（长程依赖），有的关注标点和分隔符。

5. 计算复杂度：为什么长上下文是个问题

这是你在推理部署中会碰到的问题：

操作复杂度瓶颈在哪
Q·KᵀO(n² · d)n=seq_len，二次增长！
softmaxO(n²)每个 token 要对全部 n 个 token 求 softmax
·VO(n² · d)加权平均也是 O(n²)

这就是为什么 128K 上下文的推理很贵——attention 的计算量随序列长度平方增长。FlashAttention 通过分块计算（tiling）和重计算（recomputation）把显存从 O(n²) 压到了 O(n)，但计算量还是 O(n²)。

6. Self-Attention vs Cross-Attention vs Causal Attention

类型Q 来源K,V 来源用途
Self-Attention 当前序列当前序列 Encoder / Decoder 自己内部
Cross-Attention DecoderEncoder 输出 原始 Transformer decoder 看 encoder
Causal Attention 当前序列当前序列 + mask GPT 自回归生成（不能偷看未来）

Causal Attention 的实现很简单——在 softmax 之前把未来位置的分数设为 -∞：

scores = Q @ Kᵀ / √dₖ
scores = scores.masked_fill(causal_mask == 0, float('-inf'))
# 位置 i 只能看到 [0, i]，看不到 [i+1, n]
attn = softmax(scores) @ V

7. 自检

问题 1/4

为什么 Self-Attention 使用 Q·Kᵀ/√dₖ 而不是直接用 Q·Kᵀ？

A. 为了减少计算量
B. √dₖ 是一个经验常数，让数值更稳定但去掉也能训
C. 防止点积值过大导致 softmax 梯度消失——当 dₖ 很大时点积方差会增大，除以 √dₖ 保持方差稳定
D. 为了让 Q 和 K 的模长相等

问题 2/4

Multi-Head Attention 中，如果 d_model=768，h=12，每个头的 dₖ 是多少？做完 attention 后 concat 的向量维度是多少？

A. dₖ=64, concat=768
B. dₖ=64, concat=768（但需要再过 W_O 投影才是最终输出）
C. dₖ=768, concat=9216
D. dₖ=12, concat=768

问题 3/4

Causal Attention 的 mask 为什么放在 softmax 之前而不是之后？

A. 放在 softmax 之后会改变梯度的符号
B. 放在 softmax 之前效率更高
C. 放在 softmax 之后 mask，剩余位置的权重和不为 1，破坏了概率分布；放在之前设为 -∞，softmax(-∞)=0，剩余位置自动归一化
D. PyTorch 的 softmax 不支持 masked_fill

问题 4/4

训练时，一个 batch 里有 32 个句子，每个句子 128 个 token，d_model=512。Q 的形状是什么？Q·Kᵀ 的结果形状是什么？

A. Q:(32,512,128), QKᵀ:(32,512,512)
B. Q:(32,128,512), QKᵀ:(32,128,128)
C. Q:(128,32,512), QKᵀ:(128,128,32)
D. Q:(32,512), QKᵀ:(32,128)
