第4课：GPT 架构详解

从 Token IDs 输入到 Next-Token Probability 输出，逐层走一遍完整的前向传播
预计时间：40 分钟  |  2026-06-17

1. GPT 和原始 Transformer 的关系

原始 Transformer（2017）有 Encoder 和 Decoder 两部分。GPT 只保留了 Decoder，去掉了 Cross-Attention（因为没有 Encoder 可以 cross）。

原始 Transformer: GPT: Encoder Decoder Decoder-only ┌──────┐ ┌──────┐ ┌──────────────┐ │Self │ │Causal│ │Causal Attn │ │Attn │ │Attn │ │ + │ │ + │ → │ + │ │Pre-LayerNorm │ │FFN │ │Cross │ │ + │ └──────┘ │Attn │ │SwiGLU FFN │ │ + │ │ + │ │FFN │ │Residual │ └──────┘ └──────────────┘ × N 层

去掉 Cross-Attention 意味着：GPT 生成每个 token 时，只能看到自己之前生成的 token（以及 prompt），不能像翻译模型那样回看源语言的编码。

2. 完整架构：从输入到输出

以 GPT-2 Small 为例（124M 参数），逐层跟踪一个 4-token 输入的前向传播：

输入： "央行 降息 利好 债市" ↓ tokenizer.encode() [1234, 5678, 9012, 3456] ← token IDs, shape: (1, 4)

Step 1: Token Embedding

wte = nn.Embedding(vocab_size=50257, embedding_dim=768)

token_ids: (1, 4)
  → wte(token_ids): (1, 4, 768)

每个 token ID 查表得到一个 768 维向量。
这个表是训练出来的——语义相近的词，向量也相近。

Step 2: Positional Encoding

Attention 本身没有顺序概念——"央行降息"和"降息央行"在 Attention 眼里没有区别（都是两两算相关度）。所以必须注入位置信息。

wpe = nn.Embedding(max_seq_len=1024, embedding_dim=768)

positions = [0, 1, 2, 3]
pos_emb = wpe(positions): (1, 4, 768)

x = wte(token_ids) + wpe(positions)    ← 直接相加！

GPT-2 用的是可学习的位置编码（Learned Positional Embedding）。原始 Transformer 用的是正弦位置编码（Sinusoidal），后来 GPT-3/LLaMA 用了 RoPE（旋转位置编码），这个后面再讲。

关键设计决策：Token Embedding 和 Position Embedding 直接相加而不是拼接。为什么？拼接会让维度膨胀（768+768=1536），下游所有层都要跟着翻倍。相加保持维度不变，同时实验证明模型能学会在 768 维空间里把"语义"和"位置"信息编码到不同的子空间。

Step 3: Transformer Block（重复 N 次）

每个 Block 包含两个子层：Causal Self-Attention + FFN。GPT-2 使用 Pre-Norm（LayerNorm 放在子层之前）：

def transformer_block(x):
    # 子层 1: Causal Self-Attention
    x = x + causal_self_attention(ln_1(x))   ← Pre-Norm + Residual

    # 子层 2: Feed-Forward Network
    x = x + ffn(ln_2(x))                      ← Pre-Norm + Residual

    return x
Pre-Norm vs Post-Norm：原始 Transformer 用的是 Post-Norm（x = ln(x + sublayer(x))），但训练不稳定，需要 warmup。Pre-Norm（x = x + sublayer(ln(x))）让梯度在残差路径上无损传播，训练更稳定，不需要精心调 warmup。所有现代 LLM 都用 Pre-Norm。

两个子层拆开看：

子层 1: Causal Self-Attention

# 从 768 维投影到 Q/K/V 空间
Q = ln_1(x) @ W_Q    # (1, 4, 768) → (1, 4, 768)
K = ln_1(x) @ W_K    # 上节课讲过
V = ln_1(x) @ W_V

# Multi-head: 切成 12 个头
Q = Q.view(1, 4, 12, 64).transpose(1, 2)  # → (1, 12, 4, 64)
K = K.view(1, 4, 12, 64).transpose(1, 2)
V = V.view(1, 4, 12, 64).transpose(1, 2)

# Scaled dot-product attention + causal mask
attn = softmax(Q @ Kᵀ / √64 + causal_mask)  # (1, 12, 4, 4)
out  = attn @ V                               # (1, 12, 4, 64)

# 拼回头，过输出投影
out = out.transpose(1, 2).reshape(1, 4, 768)  # → (1, 4, 768)
out = out @ W_O                                 # (1, 4, 768)

子层 2: Feed-Forward Network

GPT-2 的 FFN 很简单，就是第 2 课讲的两层 MLP：

# GPT-2 FFN: Linear → GELU → Linear
h = ln_2(x) @ W_fc     # (1, 4, 768) → (1, 4, 3072), 4× expansion
h = gelu(h)             # GELU 激活（比 ReLU 平滑）
out = h @ W_proj        # (1, 4, 3072) → (1, 4, 768), 投影回来

现代 LLM（LLaMA 等）把 GELU 换成了 SwiGLU（两个线性投影 + SiLU + 逐元素乘），表达能力更强但参数量也多了 1/3。

Step 4: 语言模型头（LM Head）

N 个 Block 之后，最后的 hidden state 过一层 LayerNorm，然后投影到词表大小：

# x shape: (1, 4, 768)   ← N 个 Block 后的输出
x = ln_f(x)
logits = x @ wte.weight.T    # (1, 4, 768) @ (768, 50257) → (1, 4, 50257)
                               # 注意：GPT-2 用 wte 的转置做 LM head（weight tying）
probs = softmax(logits[:, -1, :])  # 只取最后一个位置 → (50257,)
                                      # 这就是 "next token" 的概率分布
Weight Tying：GPT-2 的 LM Head 权重 = Token Embedding 权重的转置。输入 embedding 和输出投影共享同一套参数，节省了 50257×768 ≈ 38M 参数。这个设计在中小模型里常见，大模型（LLaMA-7B+）通常分开。

3. 一页纸总结：GPT-2 Small 完整规格

关键数字的内在关系：

d_model = 768 ← 这个数字决定了整个网络的"带宽" 为什么是 768？ d_model = h × dₖ = 12(头数) × 64(每头维度) = 768 为什么 FFN 是 3072？ FFN_hidden = 4 × d_model = 4 × 768 = 3072 ← 标准 4× 扩展 为什么每层 Attention 有 4 个 768² 矩阵？ W_Q + W_K + W_V + W_O = 4 个投影矩阵，每个都是 768×768 → Attention 部分: 4 × 768² ≈ 2.36M 为什么每层 FFN 有 8 个 768² 的等效参数量？ W_fc: 768 × (4×768) = 4 × 768² W_proj: (4×768) × 768 = 4 × 768² → FFN 部分: 8 × 768² ≈ 4.72M 每层总计: 4×768² + 8×768² = 12 × 768² = 12 × 589,824 ≈ 7.1M 12 层: 12 × 7.1M ≈ 85M 词表 Embedding 为什么这么大？ 50257 个 token，每个 768 维 → 50257×768 ≈ 38.6M → 占了总参数量的 31%，但只做查表操作

4. 训练 vs 推理：同一个 forward 的不同玩法

方面训练推理（自回归生成）
输入 完整序列 [t₁, t₂, ..., tₙ] 逐 token 生成
输出 每个位置都预测下一个 token
（teacher forcing）		 只取最后一个位置
采样/贪心选 next token → 拼回去
loss 所有位置的 cross-entropy 不需要算 loss
mask Causal mask（同一批内所有 seq） Causal mask
KV Cache 不需要（并行算整条序列） 必须用！已生成的 K/V 存着，
新 token 只算自己的 Q，避免重复计算
KV Cache 是你部署推理时必须理解的东西：推理时第 t 步生成 token，前 t-1 步的 K 和 V 已经在之前的 attention 里算过了。如果不缓存，每生成一个新 token 都要把整条序列重算一遍 attention，复杂度从 O(n) 退化到 O(n²)。KV Cache 把每层的 K、V 存起来，新 token 只算自己的 Q，然后和缓存的 K、V 做 attention。代价是显存占用 = 2 × n_layers × d_model × seq_len（FP16 下 1K token 大约几百 MB，32K 就几十 GB 了）。

5. 自检

问题 1/4

GPT-2 为什么用 Pre-Norm（LayerNorm 放在子层之前）而不是原始 Transformer 的 Post-Norm？

A. Pre-Norm 计算量更小
B. Pre-Norm 让梯度通过残差连接无损传播，训练更稳定，不需要精心设计 warmup
C. Pre-Norm 可以让模型层数加倍
D. Post-Norm 已被证明会导致过拟合

问题 2/4

Token Embedding 和 Position Embedding 相加而不是拼接，主要原因是什么？

A. 相加比拼接计算更快
B. 拼接会让词表大小翻倍
C. 拼接会让隐藏维度膨胀，下游所有层的参数量都跟着平方增长；相加保持维度，模型能在同一空间内解耦语义和位置信息
D. 实验证明拼接效果差

问题 3/4

GPT-2 Small 的 hidden_size=768。Q 和 K 的每个 head 维度 dₖ=64。Q·Kᵀ 之前除以 √64=8。如果 dₖ 变成 128，不缩放的情况下，Q·Kᵀ 的方差大约会怎么变化？（假设 Q 和 K 的各分量独立同分布，均值为 0，方差为 1）

A. 方差不变
B. 方差变成原来的 1/128
C. 方差变成原来的 128 倍——点积是 dₖ 个独立随机变量的和，每个方差为 1，和为 dₖ。除以 √dₖ 正好把方差拉回 1
D. 方差变成原来的 √128 倍

问题 4/4

推理时如果不使用 KV Cache，生成第 t 个 token 时，attention 层需要计算哪些 Q·Kᵀ 点积？（假设总长度 t=1000）

A. 只需要新 token 对前 999 个 token 的点积，1000-1=999 次
B. 需要重新计算全部 1000×1000 次点积——因为历史 token 的 Q 和 K 都重新算了一遍。加上前 999 步的累积，总计算量是 O(n²) 而不是 O(n)
C. 需要计算全部 1000×1000 次点积，但前面算过的可以复用，实际只算 1000 次
D. 不需要计算，因为 Causal Mask 已经把未来 token 屏蔽了
