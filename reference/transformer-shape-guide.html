形状变化总结

[4] Token IDs ↓ Embedding 查表 [4, 768] Embedding ↓ + Position Encoding [4, 768] x ↓ QKV 投影 (×3) → 分头 [12, 4, 64] Q, K, V（每个 head） ↓ Q·Kᵀ: [12,4,64] @ [12,64,4] [12, 4, 4] Attention Scores（token 间交互的唯一地方） ↓ softmax → @ V: [12,4,4] @ [12,4,64] [12, 4, 64] Head Outputs ↓ concat + W_O [4, 768] Attn Output ↓ + Residual → LN [4, 768] ↓ W_fc: 768→3072 → GELU → W_proj: 3072→768 [4, 768] FFN Output ↓ + Residual → LN (×12 层) [4, 768] Final Hidden ↓ WTEᵀ: 768→50257 [4, 50257] Logits ↓ softmax(最后一行) [50257] Next Token 概率

核心规律：seq_len(4) 只在 Q·Kᵀ 这一步出现过 (4,4) 形状——那是唯一 token 间交互的地方。 其余所有步骤（QKV 投影、FFN、LN、LM Head）都是"每个 token 独立计算"，第 0 个 token 和第 3 个 token 互不干涉。
