三、关键规律

1. 参数量由 d_model 决定，不是层数

GPT-2 Small: N=12, d_model=768 → 124M
GPT-2 XL: N=48, d_model=1600 → 1.5B (层数 ×4, 参数 ×12)
LLaMA-7B: N=32, d_model=4096 → 6.7B (层数 2.7×GPT-2 Sm, 参数 ×54)

N 的增长是线性的（每层 ≈ 12×d_model²），d_model 的增长是二次的。d_model 翻倍，每层参数 ×4。

2. 词表占比随 d_model 增大趋近于 0

词表参数 ∝ d_model，Block 参数 ∝ d_model²。d_model 从 768 涨到 4096 时：Block 参数 ×(4096/768)² ≈ ×28，词表 ×5.3。差距只会越来越大。

3. dₖ 几乎不变（64 或 128），头数随 d_model 线性增长

GPT-2 Small: dₖ=64, h=12。LLaMA-7B: dₖ=128, h=32。每头维度选 64~128 是经验最优区间——太小表达能力差，太大 attention 计算变慢。所以 d_model 变大时，不扩 dₖ，而是加头数。

4. FFN 扩展比在缩小

GPT-2/3: 标准 4×。LLaMA-7B 引入 SwiGLU 后等效 ≈ 2.67×(但总参数量因为有 3 个矩阵而接近 4×时的 3/2 → 实际仍约 8×d_model²)。LLaMA-3-405B 降到 3.25×。大模型倾向用更低的扩展比，把参数量分配给更多的层。

5. MoE 改变了"激活参数"和"总参数"的关系

Mixtral 8×7B：总参数 46.7B，但每 token 只激活 12.9B（7B Dense 的 1.8×），推理速度和 13B Dense 模型差不多。
